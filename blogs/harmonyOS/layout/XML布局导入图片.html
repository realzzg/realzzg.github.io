# Java

在entry/src/main/java/你的包名/slice/下，新建ImageSlice.java

默认代码如下：

package com.example.myapplication.slice;

public class ImageSlice {
}

加入以下代码

package com.example.myapplication.slice;

import com.example.myapplication.ResourceTable;
import ohos.aafwk.content.Intent;
import ohos.agp.components.*;
import ohos.agp.utils.LayoutAlignment;

public class ImageSlice extends MainAbilitySlice{

    @Override
    public void onStart(Intent intent) {
        super.onStart(intent);

        //两种声名布局的方式
        DirectionalLayout directionalLayout = new DirectionalLayout(this);
        //DirectionalLayout directionalLayout = new DirectionalLayout(getContext());
        directionalLayout.setWidth(DirectionalLayout.LayoutConfig.MATCH_PARENT);
        directionalLayout.setHeight(DirectionalLayout.LayoutConfig.MATCH_PARENT);
        //设置组件垂直居中
        directionalLayout.setAlignment(LayoutAlignment.VERTICAL_CENTER);
        //设置布局属性
        directionalLayout.setOrientation(Component.VERTICAL);

        //新建Image组件
        Image image = new Image(getContext());
        //读取相对路径下的图片
        image.setPixelMap(ResourceTable.Media_icon);

        //设置布局属性

        //宽度 高度
        image.setHeight(DirectionalLayout.LayoutConfig.MATCH_CONTENT);
        image.setWidth(DirectionalLayout.LayoutConfig.MATCH_CONTENT);
        //第二种设置布局属性
        //DirectionalLayout.LayoutConfig imageLayoutConfig = new DirectionalLayout.LayoutConfig(ComponentContainer.LayoutConfig.MATCH_CONTENT, ComponentContainer.LayoutConfig.MATCH_CONTENT);
        DirectionalLayout.LayoutConfig imageLayoutConfig = new DirectionalLayout.LayoutConfig();
        //设置水平居中
        imageLayoutConfig.alignment = LayoutAlignment.HORIZONTAL_CENTER;
        image.setLayoutConfig(imageLayoutConfig);

        directionalLayout.addComponent(image);

        //添加到组件树
        super.setUIContent(directionalLayout);
    }
}

在代码中，可以看到设置组件大小的方式有两种：

  • 通过setWidth/setHeight直接设置宽高
  • 通过setLayoutConfig方法设置布局属性来设定宽高

第二种方法可以有更多的布局属性，也可以混写两种方法来缩小代码长度

它们的取值一致，有以下取值：

  • 具体数值
  • MATCH_PARENT：表示组件大小将扩展为父组件允许的最大值，它将占据父组件方向上的剩余大小。
  • MATCH_CONTENT：表示组件大小与它内容占据的大小范围相适应。
